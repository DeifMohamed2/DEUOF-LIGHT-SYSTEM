# DEUOF LIGHT System

نظام مخزون ومبيعات وبيانات أسعار باستخدام **Node.js** و **Express** و **EJS** و **MongoDB** (هيكل MVC).

## المتطلبات

- Node.js 18+
- MongoDB يعمل محلياً أو عنوان اتصال Atlas

## التشغيل

1. انسخ البيئة:

   ```bash
   cp .env.example .env
   ```

   عدّل `MONGODB_URI` و`SESSION_SECRET` حسب بيئتك.

2. ثبّت الحزم:

   ```bash
   npm install
   ```

3. أنشئ مستخدمي الدخول الافتراضيين والإعدادات:

   ```bash
   npm run seed
   ```

   - **مدير:** `admin` / `Admin@123`
   - **موظف:** `user` / `User@123`

   غيّر كلمات المرور بعد أول تشغيل.

4. شغّل التطبيق:

   ```bash
   npm start
   ```

   افتح **http://** وليس **https://** (مثلاً [http://localhost:3000](http://localhost:3000)). إذا فتحت `https://` والخادم يعمل على HTTP فقط يظهر خطأ مثل `ERR_SSL_PROTOCOL_ERROR`.

   إذا كان `NODE_ENV=production` وتتصفح عبر HTTP، قد لا تُحفظ الجلسة ما لم تضبط `SESSION_COOKIE_SECURE=false` أو تضع التطبيق خلف بروكسي HTTPS مع `TRUST_HTTPS_PROXY=true` و`SESSION_COOKIE_SECURE=true` في `.env`.

## المزايا

- **المخزن:** إضافة وتعديل الأصناف (الحذف للمدير فقط)، بحث فوري عبر `/api/inventory/search`.
- **بيانات الأسعار:** طلب تسعير (كود، اسم، كمية) أو عرض سعر من المخزون (مع سعر)، حفظ دائم، تصدير **PDF** (اسم الشركة والشعار) أو **Excel**.
- **المبيعات:** تسجيل البيع مع خصم الكمية من المخزن؛ المدير يرى كل المبيعات، الموظف يرى مبيعاته.
- **الإيرادات:** تبويب وتقرير للمدير فقط مع فلترة بالتاريخ ومخطط شهري.
- **إعدادات الشركة:** للمدير — اسم، عنوان، هاتف، شعار للـ PDF.

## الأمان (مبسّط لاستخدام محلي)

- جلسات في MongoDB، تقييد محاولات تسجيل الدخول على `/login`، رؤوس Helmet خفيفة (بدون CSP صارم). **لا يوجد CSRF** — مناسب لاستخدام شخصي/شبكة موثوقة.

## التطوير

```bash
npm run dev
```

يستخدم `node --watch` لإعادة التشغيل عند تغيير الملفات.

## PDF

يعتمد التصدير على **Puppeteer** (يحمّل Chromium عند التثبيت). على خوادم Linux قد تحتاج تبعيات إضافية لنظام التشغيل؛ راجع [توثيق Puppeteer](https://pptr.dev/guides/system-requirements).
